# Листория — Design System

> Этот документ является единственным источником истины для визуальной части проекта.

Все страницы, компоненты, цвета, размеры, отступы и анимации должны соответствовать этому документу.

---

# 1. Design Philosophy

## Главная идея

Интерфейс должен выглядеть как современный цифровой продукт уровня:

- Linear
- Notion
- Arc Browser
- Discord
- Steam
- Apple

При первом открытии пользователь должен почувствовать:

- качество;
- аккуратность;
- современность;
- плавность;
- простоту.

---

## Основные принципы

### Минимализм

Каждый элемент существует только если он действительно нужен.

---

### Воздух

Большое количество свободного пространства.

Интерфейс не должен выглядеть перегруженным.

---

### Контраст

Основной акцент — контент.

Интерфейс не должен отвлекать пользователя от произведений.

---

### Глубина

Карточки должны ощущаться отдельными слоями.

Использовать:

- тени;
- blur;
- прозрачность;
- небольшие свечения.

---

### Плавность

Все взаимодействия сопровождаются небольшой анимацией.

Интерфейс должен ощущаться "живым".

---

### Единый стиль

Любой новый компонент обязан выглядеть так, будто существовал в проекте с самого начала.

# 61. Motion System

Анимации должны быть практически незаметными.

Пользователь должен ощущать плавность интерфейса, а не видеть сами анимации.

Основной принцип:

> Motion follows purpose.

Если анимация не улучшает UX — она не нужна.

---

## Animation Duration

| Token | Duration |
|--------|----------|
| instant | 0ms |
| fast | 120ms |
| normal | 200ms |
| smooth | 300ms |
| slow | 450ms |

---

## Timing Function

По умолчанию использовать

```css
ease
```

или

```css
cubic-bezier(.4,0,.2,1)
```

---

## Hover Animation

Используется для:

- кнопок
- карточек
- sidebar
- dropdown

Продолжительность

```text
200ms
```

---

## Page Transition

Переход между страницами

```text
Fade

Opacity

TranslateY 8px
```

Продолжительность

```text
250ms
```

---

## Card Animation

При появлении

```text
Opacity

0 → 1

TranslateY

16px → 0
```

---

## Modal Animation

Открытие

```text
Fade

Scale

0.96 → 1
```

---

## Sidebar Animation

Desktop

Без анимации.

Mobile

```text
Slide Left
```

---

## Search Animation

При фокусе

- появляется glow;
- border становится Primary;
- раскрываются результаты поиска.

---

## Hero Animation

Разрешается:

- плавное появление текста;
- небольшое движение иллюстрации;
- лёгкий gradient shift.

Запрещается:

- параллакс больших масштабов;
- резкие движения;
- сложные 3D-анимации.

# 63. Mobile Guidelines

На мобильных устройствах:

Hero становится ниже.

Sidebar скрывается.

Карточки переходят в одну колонку.

Кнопки становятся выше.

Размер текста увеличивается.

Расстояние между элементами увеличивается.

# 65. Theme Switching

Переключение темы должно происходить мгновенно.

Без перезагрузки страницы.

Использовать CSS Variables.

Не использовать отдельные стили для каждого компонента.

Все цвета должны браться из Theme Tokens.

---

# 66. Design Tokens

Никогда не использовать внутри компонентов:

```tsx
className="bg-[#8B5CF6]"
```

Использовать только:

```tsx
bg-primary

text-primary

bg-card

bg-background

border-border
```

Все токены централизованы.

---

# 67. UI Consistency Rules

Каждый новый компонент обязан:

- использовать существующие токены;
- использовать существующие размеры;
- использовать существующие радиусы;
- использовать существующие тени;
- использовать существующие анимации.

Если существующего компонента недостаточно — сначала расширить Design System.

Запрещено создавать случайные стили.

---

# 68. Component Creation Rules

Перед созданием нового компонента необходимо проверить:

- существует ли аналог;
- можно ли переиспользовать существующий;
- нарушает ли компонент единый стиль.

Любой новый компонент должен выглядеть так, будто он существовал в проекте с первого дня.

---

# 69. Codex Design Rules

Перед созданием любого интерфейса Codex обязан:

1. Прочитать PROJECT.md.
2. Прочитать DESIGN_SYSTEM.md.
3. Использовать только Design Tokens.
4. Использовать только существующие компоненты.
5. Не придумывать новые цвета.
6. Не использовать случайные размеры.
7. Не использовать inline styles.
8. Поддерживать Light и Dark Theme.
9. Использовать TailwindCSS.
10. Использовать shadcn/ui как базовую библиотеку.

# 71. Non-negotiable Rules

- Использовать только TypeScript.
- Использовать только TailwindCSS.
- Использовать только App Router.
- Использовать только shadcn/ui.
- Использовать только Lucide Icons.
- Использовать только Design Tokens.
- Поддерживать Dark Theme.
- Поддерживать Light Theme.
- Не использовать случайные цвета.
- Не использовать случайные размеры.
- Не использовать большие компоненты (>300 строк без причины).
- Не смешивать UI и бизнес-логику.
- Не создавать новый стиль, если уже существует подходящий.

- современным;
- чистым;

- лёгким;
- красивым;
- последовательным;
- масштабируемым.
